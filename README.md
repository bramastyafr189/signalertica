# 📡 Signalertica | Smart Signal Tracker

Signalertica is a real-time keyword intelligence and monitoring system. It allows you to define custom **Surveillance Channels** (interests), associate tracking keywords, scan public feeds (powered by Google News RSS) for matches, and receive instant **Web Push Notifications** directly to your browser or mobile device when new signals are detected.

---

## 🚀 Key Features

* **Custom Surveillance Channels:** Create isolated tracking feeds based on specific domains (e.g., Cyber Security, Market Trends, Crypto).
* **Granular Keyword Filtering:** Associate multiple tracking words to a single channel.
* **Smart Background Sync (Cron API):** Automate scans at user-defined intervals (e.g., every 5 mins, 30 mins, or manual).
* **Instant Web Push Alerts:** Real-time push notifications signed with VAPID keys, deep-linking directly to specific intelligence logs.
* **Auto-Purge (Self-Cleaning):** Automatic clean-up routines delete logs older than 24 hours to keep the database light and performant.
* **Premium Dashboard:** Single-page cyber-inspired dark mode UI built with responsive layout, glassmorphic accents, and smooth animations.

## 📊 Database Schema Relationships

The database is built on top of SQLite/LibSQL with the following entity relationships:

* **`interests` (Channels) ───[ 1 : N (Cascade) ]───> `keywords`:** Decommissioning a channel automatically purges all associated keywords.
* **`intelligence_logs` ───[ 1 : N (Cascade) ]───> `captured_articles`:** A scan log houses multiple matched article entries. Deleting older logs automatically cascades to clear the articles.
* **`push_subscriptions`:** Keeps track of authorized client browser push tokens.
* **`system_settings`:** Houses global status parameters (e.g., global sync kill switch).

## ⚙️ Configuration & Getting Started

### 1. Prerequisites
Ensure you have Node.js (v18+) and npm installed.

### 2. Environment Setup
Copy `.env.example` to `.env.local`:
```bash
cp .env.example .env.local
```
Fill in the parameters:
* **Database:** If `TURSO_CONNECTION_URL` is omitted, the app falls back to a local SQLite database (`file:signalertica.db`).
* **VAPID Keys:** Generate push notification credentials:
  ```bash
  npx web-push generate-vapid-keys
  ```
  Paste the generated public and private keys into `NEXT_PUBLIC_VAPID_PUBLIC_KEY` and `VAPID_PRIVATE_KEY`.
* **Cron Secret:** Set a custom secure string for `CRON_SECRET` to protect production sync calls.
* **Admin Emails:** Set `ADMIN_EMAILS` to a comma-separated list of Google account emails that may change global sync settings.
* **Production Auth:** `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `NEXTAUTH_SECRET`, and `CRON_SECRET` must be real, secret production values.

### 3. Database Migration
Initialize or upgrade your database using the checked-in Drizzle migrations:
```bash
npx drizzle-kit migrate
```

### 4. Running the App
Start the development server:
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) to access your Signalertica dashboard.

### 5. Automated Scanning (Production)
To automate keyword checks in production, configure a scheduled HTTP GET job (e.g., every 10 minutes) targeting:
```text
GET https://your-domain.vercel.app/api/cron/scan
Header: Authorization: Bearer <YOUR_CRON_SECRET>
```
